Ce repository est axé sur la prédiction de la probabilité qu'un client présente concernant son incapacité à rembourser son prêt. Il est structuré comme suit :

- CODE : Dossier renfermant plusieurs éléments essentiels :
	- Fichier Introductif.txt : Texte décrivant la structure du dépôt.
	- API : Inclut les fichiers api.py et test_unitaire.py, ainsi que les fichiers nécessaires au déploiement sur le cloud
	- Dashboard : Comprend le script dashboard, les resources (modèles, data, pickle) et les requirements. 
	- Data-drift : Intègre le script pour évaluer le Data-Drift avec evidently et un fichier HTML présentant les résultats de cette analyse.
	- Notebooks : Englobe les notebooks nécessaires à l'élaboration du jeu de données final et à l'évaluation de différents modèles de classification (EDA, Pre-processing, Feature Selection, Modelling, Dashboard)
	- MLRuns : Rassemble tous les enregistrements MLflow relatifs aux modèles de classification.
